## Specialist Flags
- **WSL argv passing has TWO interacting quoting layers and is content-dependent** (Windows `list2cmdline` framing + WSL's `bash -c` reconstruction). A no-space value with parens (`‚Ä¶/Python_(programming_language)`) raw ‚Üí `bash` syntax error; quoted ‚Üí arrives clean. A value with spaces ‚Üí arrives WITH literal quotes attached. This is why the decode is load-bearing and why Option B (never put untrusted values on the command line) is the sturdier design. Filed to memory `wsl-argv-quoting-two-layer`.
- **Fail-open in bash is easy to write and hard to spot.** The original decode used `< <(python3 ‚Ä¶)` process substitution + a `while read` loop ‚Äî BOTH exempt from `set -e`, so a crashed/partial decode silently ran a truncated command. The fix: plain pipeline `{ ‚Ä¶ } > file || abort`, `surrogateescape`, explicit argv-count check. When a bash change sits under a `# fail CLOSED` invariant, scrutinize process-subs and loop conditions specifically.
