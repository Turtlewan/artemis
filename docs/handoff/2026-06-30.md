## Decisions Made
- **Memory engine = Cognee (confirmed by live spike, not just on-paper).** Ran both engines live on LoCoMo using the **subscription `gpt-5.5` via a `codex exec`→OpenAI proxy** (owner chose subscription-not-API; subscriptions aren't web APIs, so a ~120-line shim bridges them) + local Ollama embeddings. Cognee wins on operational cost (in-process, ~4× faster ingest, ~4× fewer LLM calls); Graphiti's bi-temporal edge unproven-as-material at the tested scale. Full verdict: `docs/findings/cognee-vs-graphiti-spike-2026-06-30.md`. Owner called the spike at the 2-session decision point to conserve quota.
- **Cognee is an OPTIONAL dependency group** (`memory`), not core — keeps the spine thin (lazy `importlib`; nothing outside `memory/` imports cognee). Memory's internal extraction/answer LLM defaults to a **small/local model** (owner-confirmed: memory grunt-work ≠ flagship reasoning; the router right-sizes per call).
- **Slice 2 built as 6 incremental specs** (v2-07..12), one at a time per the locked cadence — owner asked for "all four" deferred pieces and got them sequenced, not as a mega-commit.
- **Consolidation reasoning + summarization dogfood our own `ModelPort`** (imported via `ports.model` only, so no provider/anthropic pull) — memory uses the subscription-first router for its ADD/UPDATE/DELETE/NOOP classification and overflow summaries.
- **Decay metadata = durable SQLite ledger** (owner chose over in-memory) — per-fact recency/access/salience survives restart; `forget()` archives, never deletes.

## Specialist Flags (gotchas git diff won't surface)
- **Live smokes caught two real bugs the hermetic (fake-cognee) tests could not** — keep smoking the real engine: (1) `SearchType.CHUNKS` returns node **dicts**, not strings → `_extract_text` pulls the `text` field (else context was serialized-dict garbage); (2) the first consolidating `write` searches an **uninitialized** Cognee store → `_similar` is now best-effort (degrades to ADD). Also (3) supersession used **exact** content match but the LLM dropped a trailing period → normalized matching (case + trailing punctuation). Paraphrase-level supersede matching is still NOT handled (future: embedding similarity).
- **`codex exec` ≈ 13k tokens overhead per call** — fine for eval/occasional use, **not** for high-volume memory extraction in production (use local Ollama there). The proxy shim (`scratch/spike/codex_proxy.py`) is a reusable pattern but throwaway here.
- **Background-task reaper** kills long local runs (cognify is slow on the 8GB box) — the build/verify is unaffected (hermetic + fast); only the live smokes fought it. Run live smokes foreground or expect retries.
