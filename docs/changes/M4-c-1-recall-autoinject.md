# Spec: M4-c-1 — Recall + auto-inject (current facts → system prompt, token-budgeted) wired into the M1 Brain turn loop + post-turn write-path scheduling + decay-ranked recall

**Identity:** Implements the memory READ/ranking/injection path on top of M4-a/M4-b: the decay scorer (`decay_score`, `rank_for_inject`, `recall_multiplier`), `inject_context` (rank current `as_of=now` facts above an inject threshold by recency × salience × access, pack into the system prompt within a token budget, bump access), decay-ranked `recall`, and wires auto-inject + the M4-b async write queue into the M1 `Brain.respond` turn loop (inject before the model call; schedule extraction after). Schema/repository = M4-a; write-path decisioning = M4-b; owner view/edit/delete/purge + the tombstone sweep = M4-c-2.
→ why: see docs/technical/adr/ADR-004-memory-engine.md (auto-inject each turn within a token budget; decay recency×salience×access, never-hard-delete) · docs/technical/architecture/brain.md § Memory (auto-inject; forgetting/decay) · docs/research/memory-engine-research.md (decay half-life tuning gated).

- **Decay score** = `recency × salience × access` (ADR-004 Ebbinghaus-style). Drafting: `recency = exp(-Δt_days / half_life_days)` (Δt from `last_access` or `valid_from`); `salience` = the stored `salience` column (default 1.0; boosted for explicit owner-stated facts — set by M4-b confidence/owner edits); `access = 1 + log1p(access_count)`. `score = recency * salience * access * confidence`. Facts with `score < inject_threshold` are NOT injected (demoted). → impact: Caution. [RESOLVED — drafted defaults, tuned at GATED Task 4: `half_life_days = 14.0`, `inject_threshold = 0.2` as config-tunable constants; they do NOT change the code shape, only the constants.]
- **Composite forgetting score (recall re-rank):** per ADR-004's brain-sweep absorption, recall on the SEMANTIC store applies the composite forgetting score `I(m,t)=α·exp(−λΔt)+β·access_count+γ·cosine(m,query)` as a retrieval-time re-rank MULTIPLIER (recently-accessed ×1.5, stale ×0.3) over sqlite-vec similarity — NOT eager deletion. The EPISODIC store stays TTL-only. Distinct from the inject-time decay score: inject ranks current facts by recency×salience×access; recall re-ranks vector candidates by the composite I(m,t) multiplier. → impact: Caution.
- **Auto-inject** (ADR-004): each turn, `inject_context(person_id, token_budget, as_of=now)` queries current (`as_of=now`) facts, ranks by decay score, filters above `inject_threshold`, and packs the top facts into a compact text block within `token_budget` (greedy by score; stop when the next fact would exceed the budget). The Brain prepends this block as a system message. Accessing a fact for injection BUMPS its `access_count` + `last_access` (so used facts decay slower — the access term). → impact: Stop (this is the auto-inject contract; the access-bump closes the recency×access loop).
- Off-hardware: everything is deterministic (FakeEmbedder, the M4-a DB fixture, a `FakeModelPort` for the Brain). **The decay half-life / inject-threshold tuning on real usage traces is the GATED Task 4.** → impact: Caution.

Simplicity check: considered a memory TOOL the model calls to fetch facts — rejected: ADR-004/brain.md lock AUTO-inject (the model never calls a memory tool; facts are packed into the system prompt each turn). This is the minimum that satisfies auto-inject + decay-ranked recall + the Brain wiring.

## Specialist Context
### Security
Auto-inject only ever runs for an **owner scope with an open keyed DB** (the wall: a locked/guest scope has no memory store → no inject). The inject block contains sensitive owner facts — it goes ONLY into the LOCAL responder's system prompt (never to a cloud model; the sensitivity/escalation gate is a separate layer and the M1 escalation path is still a stub). NEVER log the inject block / fact objects at info level. A memory-read failure degrades to "no inject" — it never breaks the turn or leaks. [FLAG for apex-security (M4 gate): confirm the inject block never reaches a cloud/escalation path; confirm no fact plaintext is logged; review the inject-block prompt-injection surface (owner's own facts are trusted, but extracted facts originate from untrusted turn content — spotlight if a fact ever derives from ingested third-party content).]

### Performance
Decay is computed **lazily at inject/recall time** (no scheduler, no background daemon in M4) — O(current facts) per turn, which is small at per-person scale. Auto-inject runs once per turn before the model call; the access-bump is a single cheap UPDATE per injected fact. The token-budget greedy pack bounds the injected context. The write path is async (M4-b) — scheduling it after the turn adds no response latency. Half-life/threshold tuning (Task 4) is the only empirical knob.
